# 安全C方言 — 配列・ポインタの安全ルール仕様

## 1. 関数パラメータにおける配列宣言

### 禁止
```c
void f(int a[]); // ❌ エラー: 引数配列構文は禁止（暗黙のポインタ退化を防ぐ）
void f(int (*a)[]);    // ✅ 可変長配列へのポインタ
void f(int (*a)[10]);  // ✅ 要素数固定配列へのポインタ
int arr[5];
int *p = arr;    // ❌ 配列からポインタへの暗黙変換禁止
int (*p)[2] = a; // ❌ 配列 → 配列ポインタへの暗黙変換禁止
int *p = &a;     // ❌ 配列全体のアドレスをポインタ型に代入禁止
int arr[5];
int *p = &arr[0];      // ✅ 要素アドレスを明示的に取得
int a[2] = {1, 2};
int (*p)[2] = &a;      // ✅ 配列全体のアドレスを同型配列ポインタに代入

int (*p_var)[] = &a; // 長さ不明（可変長）配列ポインタも許可
```c
